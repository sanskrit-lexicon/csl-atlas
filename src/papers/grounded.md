# Grounded framework

Building the analytic apparatus from MW's own data, before any external theory.

📄 **Full paper:** [paper-grounded.md in MWS docs-pass](https://github.com/sanskrit-lexicon/MWS/blob/docs-pass/papers/microanalysis/paper-grounded.md) (~8K words)

## Five core constructs

1. **Block** — a discriminable structural component (18 in MW)
2. **Slot** — an ordered position within an entry (slot architecture, not free-form)
3. **Profile** — the block-set characteristic of an article type (14 in MW)
4. **Hedge** — a block whose function is to modify the entry's evidentiary status (`<ls>L.</ls>` is the singular MW hedge)
5. **Infrastructure** — blocks supporting processability, not content (F17 machine annotation, F18 correction records)

## Central thesis: block economy

MW reuses a **6-block kernel** across 286,561 entries. Type-driven enrichments add to this kernel sparingly. This is **block economy** — the design discipline of a print dictionary that cannot afford to elaborate every block in every entry.
